# 9-lite 운용 매뉴얼

경로: `projects/project_001/runtime/9lite_runbook.md`

## 1. 9-lite 개요

9-lite는 기존 9번 집필 프롬프트의 과부하를 줄이고, 회차별 목적에 맞는 본문 생성을 안정화하기 위한 경량 운용 방식이다.

기존 9번의 주요 문제는 다음과 같다.

* **컨텍스트 과부하**
  청사진, 정본 문서, 문체 메모, 카드, 금지 규칙, 비트 시트, 샘플, 출력 규칙이 한 번에 과도하게 붙어 본문 생성 초점이 흐려졌다.

* **비트 시트 강박**
  회차마다 필요한 장면 밀도와 호흡이 다른데도, 모든 회차를 동일한 구조와 비트 수로 밀어 넣는 문제가 있었다.

* **like 샘플 차단 문제**
  like 샘플이 필수처럼 붙으면 회차 톤과 맞지 않는 문장 에너지가 섞이거나, 샘플 모방 위험이 커졌다.

* **통과 게이트 부재**
  결과물이 정본 시놉을 따랐는지, 회차 톤을 지켰는지, 금지 확장을 했는지 확인하는 중간 게이트가 약했다.

9-lite의 해결 방향은 다음과 같다.

* 정본 시놉을 먼저 읽고 회차별 운용 톤을 분리한다.
* 회차마다 핵심 5항목을 새로 정한다.
* 장면을 씬 드라마 카드 5개로 압축한다.
* 본문 생성 프롬프트는 필요한 것만 붙인다.
* like 샘플은 선택 사항으로 둔다.
* 결과물은 회차별 5항목과 씬 카드 기준으로 통과 여부를 판단한다.

## 2. 회차 본문 자동 생성 4단계

### 1단계. 정본 시놉 읽기

Codex가 먼저 해당 회차의 정본 시놉을 읽는다.

기본 대상:

```text
projects/project_001/synopsis/arc_001-010.md
```

확인 항목:

* 회차 제목
* 회차 목적
* 주요 사건
* 문체/호흡
* 감정선
* 떡밥/단서
* 다음 화 연결

정본 시놉에 없는 내용은 임의 확장하지 않는다.

### 2단계. 회차 5항목 정하기

각 회차는 같은 9-lite라도 운용 톤이 다르다.
따라서 회차별로 본문 생성 전에 5항목을 새로 정한다.

5항목은 다음 기능을 가진다.

* 회차의 문체 방향 고정
* 장면 호흡 고정
* 감정 과열 방지
* 코미디 또는 건조함의 강도 조절
* 마지막 컷의 착지 방식 고정

### 3단계. 씬 카드 5개 만들기

정본 시놉을 기준으로 씬 드라마 카드 5개를 만든다.

카드는 본문 작성용 비트 시트가 아니라, 본문이 반드시 남겨야 할 결과를 정리하는 최소 장치다.

카드 1~5는 회차 흐름을 따라 배치한다.

* 카드 1: 진입 조건
* 카드 2: 핵심 관계 또는 장치 첫 등장
* 카드 3: 반복 결 또는 생활화
* 카드 4: 조연/관계/사건 인지
* 카드 5: 현재 시점 또는 다음 화 연결

### 4단계. Codex로 본문 생성

Codex에 다음을 붙여 본문을 생성한다.

* LOCK
* 회차별 5항목
* 캐릭터 톤
* 정본 시놉 요약
* 씬 카드 5개
* 분량 가이드
* 출력 규칙
* 금지 규칙
* 선택적으로 like 샘플 1개

본문 생성 후에는 다음 기준으로 통과 여부를 본다.

* 정본 시놉을 지켰는가
* 회차별 5항목이 본문에 반영됐는가
* 씬 카드 5개의 결과가 남았는가
* 이름·사연·캐릭터·별도 사건을 새로 만들지 않았는가
* 분량이 가이드 안에 들어왔는가
* 메타 용어가 본문에 노출되지 않았는가

## 3. 회차별 5항목 치환 원리 + 검증된 예시

9-lite의 5항목은 고정값이 아니다.
회차 목적에 따라 매번 치환한다.

### 치환 원리

회차별 5항목은 다음 순서로 정한다.

1. 정본 시놉의 회차 목적을 확인한다.
2. 문체/호흡 항목을 가장 우선한다.
3. 감정선의 금지 방향을 확인한다.
4. 주요 사건을 5개 카드로 나눈다.
5. 마지막 컷의 착지 방식을 정한다.

### 1화 검증 예시: 오배송·신생아 코미디

1화는 오배송 탄생 사건을 한 화에 다 담는 회차다.
성인 차사의 자의식과 신생아 육신의 무능이 충돌하는 코미디가 핵심이다.

1화 5항목:

```text
1. 짜증 우선
2. 실무자식 비꼼
3. 코미디 강제
4. 결구 폭발
5. 톤 에너지 유지
```

운용 기준:

* 한도윤의 기본 반응은 체념보다 짜증이다.
* 저승 행정 사고를 실무자식 비꼼으로 해석한다.
* 신생아 육신의 무능이 계속 장면을 방해한다.
* 말하고 싶은 내용과 실제 출력값의 괴리를 코미디로 쓴다.
* 마지막은 짧게 터지는 결구로 닫는다.

### 2화 검증 예시: 성장 압축·회상

2화는 영아기부터 청년기 직전까지 압축하고, 본편 현재 직전 상태를 세팅하는 회차다.
1화처럼 코미디를 강제하지 않는다.

2화 5항목:

```text
1. 건조 회상
2. 압축 몽타주
3. 짧은 컷 누적
4. 감정 과열 금지
5. 현재 시점 착지
```

운용 기준:

* 현재의 한도윤이 과거를 정리하듯 말한다.
* “내가 자라는 동안 이 정도였다” 식의 1인칭 건조 회상 톤을 따른다.
* 영아기, 감시자 첫 등장, 유아·아동기, 학생기, 청소년기~청년기를 짧게 누적한다.
* 가족 과보호를 신파로 키우지 않는다.
* 마지막은 30대 초반 현재, 안전진단 현장 출근으로 닫는다.

## 4. 씬 드라마 카드 5요소 포맷

씬 드라마 카드는 아래 5요소만 사용한다.

```text
카드 N.
- 장면 목적:
- 감정 전환:
- 반드시 남길 결과:
- 금지:
- 엔딩 컷:
```

### 장면 목적

해당 카드가 본문에서 수행해야 하는 기능을 적는다.

예:

```text
장면 목적: 감시자(차사 인턴)를 처음 등장시키고, 한도윤과 감시자의 위계가 첫 장면에서 바로 확정되게 한다.
```

### 감정 전환

장면이 시작할 때와 끝날 때의 감정 방향을 적는다.
감정 설명을 과하게 쓰지 않고, 독자가 느낄 전환만 고정한다.

예:

```text
감정 전환: 반복되는 위기와 관리 → “내가 자라는 동안 이 정도였다” 식 건조한 회상으로 안정.
```

### 반드시 남길 결과

본문에 반드시 남아야 할 정보나 상태 변화를 적는다.
이 항목이 카드의 핵심이다.

예:

```text
반드시 남길 결과: 친구 셋은 학생기부터 옆에 있던 증인이라는 위치만 박고, 깊이 추궁하지 않는 관계 결을 남긴다.
```

### 금지

해당 장면에서 확장하면 안 되는 요소를 명시한다.

반드시 금지할 확장:

* 이름 새로 만들기
* 가족 사연 새로 만들기
* 친구별 캐릭터성 새로 만들기
* 별도 사건 새로 만들기
* 정본 시놉에 없는 설정 추가
* 신파 확장
* 과한 코미디화
* 다음 화 사건 앞당기기

예:

```text
금지: 친구 이름, 각자 캐릭터성, 별도 사건을 새로 확장하지 않는다.
```

### 엔딩 컷

해당 카드가 끝나는 마지막 시각 이미지를 적는다.
본문의 장면 전환 기준점으로 사용한다.

예:

```text
엔딩 컷: 30대 초반 한도윤이 안전진단 현장으로 출근하는 일상에서 닫고, 다음 화 첫 귀신 사건 진입을 준비한다.
```

## 5. 9-lite 프롬프트 템플릿

9-lite 프롬프트는 붙이는 순서가 중요하다.

### 부착 순서

```text
[최상단]
LOCK
회차별 5항목
캐릭터 톤

[중간]
정본 시놉 요약
씬 카드 5개
분량 가이드
출력 규칙
금지 규칙

[선택]
like 샘플 1개

[후순위]
청사진
1번 §C
1번 §D
11번 §7
11번 §12
11번 §13-B
```

후순위 자료는 필요한 경우에만 일부 참조한다.
통째로 붙이지 않는다.

### 기본 템플릿

```text
역할:
너는 webnovel-promptpack 저장소 안에서 N화를 9-lite 방식으로 1회 생성한다.

작업:
- 정본 시놉을 따른다.
- 회차별 5항목을 따른다.
- 씬 드라마 카드 5개를 따른다.
- 본문만 출력한다.

LOCK:
- 정본 시놉 우선.
- 씬 카드 우선.
- 회차별 5항목 우선.
- 정본에 없는 이름, 사연, 캐릭터, 별도 사건 생성 금지.
- 메타 용어 본문 노출 금지.
- 해설, 평가, 작업 보고 출력 금지.

회차별 5항목:
1.
2.
3.
4.
5.

캐릭터 톤:
- 한도윤은 1인칭 기준으로 건조하고 시큰둥하다.
- 회차 목적에 따라 짜증, 비꼼, 건조 회상, 무심함의 비율을 조정한다.
- 감정은 설명하지 말고 컷과 반응으로 남긴다.

정본 시놉:
[해당 회차 시놉 요약]

씬 드라마 카드 5개:
[카드 1~5]

분량 가이드:
- 카카오 기준: 5,500~6,000자
- 네이버 기준: 5,000~5,500자
- ±10% 허용

금지:
- 정본에 없는 이름 생성 금지.
- 정본에 없는 가족 디테일 생성 금지.
- 정본에 없는 친구 디테일 생성 금지.
- 정본에 없는 사건 생성 금지.
- 다음 화 사건 앞당기기 금지.
- 메타 용어 본문 노출 금지.
- 프롬프트 항목명 본문 노출 금지.

출력:
N화 — [제목]

[본문]
```

## 6. like 샘플 사용 규칙

like 샘플은 필수가 아니다.
회차 톤을 안정시키는 데 필요할 때만 사용한다.

### 사용 조건

like 샘플은 아래 조건을 만족해야 한다.

* 회차 톤과 가깝다.
* 800~1,500자 분량이다.
* 1개만 붙인다.
* 본문 주제나 사건을 따라 하게 만들지 않는다.
* 문장 에너지와 호흡만 참고하게 한다.

### 생략 조건

아래 경우 like 샘플을 생략한다.

* 회차 톤이 정본 시놉만으로 충분히 명확할 때
* 성장 압축, 건조 회상, 몽타주 회차일 때
* 샘플이 1화 코미디 톤을 과하게 끌고 올 위험이 있을 때
* 샘플 문장 복제 위험이 클 때

### 모방 금지 표현 리스트

like 샘플을 붙일 경우 반드시 아래 금지를 같이 붙인다.

```text
like 샘플 사용 규칙:
- 샘플은 문장 에너지와 호흡만 참고한다.
- 샘플의 정확한 어휘를 복제하지 않는다.
- 샘플의 문장 구조를 그대로 베끼지 않는다.
- 샘플의 비유를 재사용하지 않는다.
- 샘플의 고유한 농담 방식을 그대로 옮기지 않는다.
- 샘플의 대사 리듬을 직접 복제하지 않는다.
- 샘플의 장면 구성을 따라 하지 않는다.
- 샘플 속 인물 관계를 옮기지 않는다.
- 샘플 속 사건 전개를 옮기지 않는다.
- 샘플은 회차 정본보다 우선하지 않는다.
```

### 회차별 판단

* 1화처럼 말맛과 코미디 엔진이 중요한 회차는 like 샘플 사용 가능.
* 2화처럼 건조 회상과 압축 몽타주가 중요한 회차는 like 샘플 생략 가능.
* like 샘플이 회차 정본 톤을 흐리면 사용하지 않는다.

## 7. 분량 가이드

플랫폼별 기본 분량은 다음을 따른다.

```text
카카오 기준: 5,500~6,000자
네이버 기준: 5,000~5,500자
허용 범위: ±10%
```

### 운용 기준

* 1화처럼 사건과 코미디 엔진을 한 화에 담는 회차는 상한에 가깝게 운용한다.
* 2화처럼 성장 압축 몽타주 회차는 장면 수가 많아도 각 컷을 짧게 유지한다.
* 분량 초과 시 설정 설명, 감정 설명, 장면 중복을 줄인다.
* 분량 부족 시 정본에 없는 사건을 추가하지 말고, 기존 카드의 감각 컷을 보강한다.

### 금지

* 분량을 채우기 위해 새 사건을 만들지 않는다.
* 가족·친구·감시자의 이름이나 사연을 새로 만들지 않는다.
* 다음 화 사건을 앞당겨 분량을 채우지 않는다.
* 설명문으로 설정을 길게 풀지 않는다.

## 9. 운용 모드

9-lite는 반자동에서 완전 자동으로 전환한다.

### 1단계. 반자동 모드

초기 2~3회차는 반자동으로 운용한다.

흐름:

```text
정본 시놉 확인
→ GPT가 회차 5항목 제안
→ 사용자 컨펌
→ GPT가 씬 카드 5개 제안
→ 사용자 컨펌
→ Codex로 본문 생성
→ GPT가 평가
→ 사용자 확정
```

반자동 모드의 목적:

* 회차별 5항목이 제대로 잡히는지 확인
* 씬 카드 포맷이 과확장 없이 작동하는지 확인
* Codex가 본문에서 정본을 지키는지 확인
* 분량과 톤이 안정되는지 확인

### 2단계. 완전 자동 모드

초기 검증 후에는 완전 자동으로 전환한다.

사용자 입력 예:

```text
3화 만들어줘.
```

자동 처리 흐름:

```text
정본 시놉 읽기
→ 회차 5항목 자동 산출
→ 씬 카드 5개 자동 작성
→ 9-lite 프롬프트 구성
→ Codex 본문 생성
→ 검수 기준으로 통과 여부 확인
```

완전 자동 전환 조건:

* 2~3회차 연속으로 정본 위반이 없다.
* 회차별 톤 치환이 안정적이다.
* 씬 카드가 과확장을 막고 있다.
* 분량이 플랫폼 가이드 안에 들어온다.
* 메타 용어가 본문에 노출되지 않는다.

## 10. 실패 시 대응

### 톤 어긋남

증상:

* 1화가 체념 위주로 흐름
* 2화가 코미디/말맛 회차처럼 흐름
* 회차 목적과 다른 감정선이 커짐

대응:

```text
회차별 5항목을 재설정한다.
프롬프트 최상단에 5항목을 다시 붙인다.
해당 회차의 문체/호흡 메모를 더 앞쪽에 둔다.
```

예:

```text
2화는 코미디/말맛 회차가 아니다.
건조 회상, 압축 몽타주, 짧은 컷 누적을 우선한다.
```

### 디테일 과확장

증상:

* 친구 이름이 새로 생김
* 감시자 이름이 생김
* 가족 구성원 사연이 확장됨
* 정본에 없는 별도 사건이 추가됨

대응:

```text
씬 카드의 금지 슬롯을 강화한다.
정본에 없는 이름, 사연, 캐릭터, 별도 사건 생성 금지를 카드별로 반복한다.
```

예:

```text
금지:
- 친구 이름 생성 금지.
- 감시자 이름 생성 금지.
- 가족 구성원 사연 생성 금지.
- 별도 사건 추가 금지.
```

### 분량 초과

증상:

* 장면이 길어진다.
* 설명이 늘어난다.
* 감정 묘사가 반복된다.
* 몽타주가 개별 사건처럼 확장된다.

대응:

```text
분량 가이드를 재명시한다.
각 카드의 장면 수를 제한한다.
설명보다 결과를 남기게 한다.
```

예:

```text
각 카드는 1~2개 컷으로만 처리한다.
설정 설명을 늘리지 말고 결과만 남긴다.
```

### 분량 부족

증상:

* 카드 결과가 빠진다.
* 장면 전환이 너무 급하다.
* 현재 시점 착지가 약하다.

대응:

```text
새 사건을 추가하지 않는다.
기존 카드의 감각 컷과 반응 컷을 보강한다.
```

예:

```text
정본에 없는 사건을 추가하지 말고, 카드 3의 가족 과보호 컷과 카드 5의 현재 진입 컷을 보강한다.
```

### 메타 용어 본문 노출

증상:

* 본문에 “5항목”, “씬 카드”, “정본 시놉”, “프롬프트”, “몽타주” 같은 작업어가 노출된다.

대응:

```text
출력 규칙을 재명시한다.
본문만 출력하게 한다.
작업 보고, 해설, 평가를 금지한다.
```

예:

```text
출력:
- 본문만 출력한다.
- 해설, 평가, 작업 보고를 붙이지 않는다.
- 프롬프트 용어를 본문에 노출하지 않는다.
```
