### 이번 단계에서 확정한 중심 방향

- 단순한 생계형 재도전물이 아니라, 플랫폼 최상위권을 정면으로 노리는 작가 회귀 성공물로 간다.
- 성공의 쾌감은 표절·기만·경쟁자 짓밟기가 아니라, 좋은 기획·성실한 연재 운영·독자 반응에 대한 정직한 대응에서 나온다.
- 회귀 지식은 정답지가 아니라 실패 원인 재검증의 출발점으로만 사용한다.
- 독자 반응과 성과 지표는 숫자 중계가 아니라, 선택의 압박·보상·관계 변화로 작동해야 한다.

## §A. 신규 산출물 — 상업 콘셉트 카드

### §A-1. 5번 세계관 설계 프롬프트용 카드 (→ 5번 직접 사용)

```markdown
- 한 줄 판매 포인트: 망한 편집자 출신 작가가 회귀해, 전생에 독자를 얕보다 망친 작품을 독자 반응과 정면승부로 다시 써 플랫폼 탑티어까지 뒤집는다.
- 핵심 독자 쾌감:
  · 지표 역전 쾌감 — 악플, 낮은 조회수, 선작 정체를 기획·수정·연재 운영으로 뒤집는 순간의 즉각 보상.
  · 회귀 보상 쾌감 — 전생의 실패 장면이 이번 생의 선택과 실력으로 반전될 때 생기는 재도전 보상.
  · 직업 역량 증명 쾌감 — 편집자 경험, 시장 감각, 원고 구조 개선, 마감 운영이 실제 성과로 검증되는 쾌감.
  · 독자존중 감정 보상 — 독자를 숫자가 아니라 기다리는 사람으로 받아들이며, 첫 고정 독자/댓글러의 반응이 주인공을 바꾸는 누적 보상.
- 플랫폼 적합성: 문피아식 실시간 연재 지표 압박과 카카오페이지식 남성향 직업 성공물 상승감을 결합해, 조회수·선작·댓글·추천·유료 전환·편집자 반응이 매 선택의 대가로 작동한다.
- 장르 공식: 회귀한 실패자가 전생의 패배 원인을 알고 다시 도전하되, 치트 한 방이 아니라 직업 역량·시장 검증·연재 경쟁을 통과하며 상승한다.
- 차별화 포인트:
  · 표절 없는 재도전 — 전생 실패작을 다시 다루지만, 남의 것을 훔치거나 독자를 속이는 성공 전략은 쓰지 않는다.
  · 독자 반응의 관계화 — 댓글과 지표를 숫자판이 아니라 주인공의 오만함을 깨고 작품을 고치게 만드는 사람의 반응으로 쓴다.
  · 편집자 출신 작가의 이중 시야 — 편집자식 분석력과 실패 작가의 절박함이 충돌해, 장면마다 계산과 진심 사이의 선택을 만든다.
  · 탑티어 목표의 압박 — 목표를 중박·생계 회복에 두지 않고 플랫폼 최상위권으로 잡아 생활비, 마감, 경쟁 선점, 유료 전환 압박을 키운다.
- 절대 양보 못하는 한 끗: 표절하지 않고, 독자를 속이지 않고, 후배 작가를 짓밟지 않으며, 독자가 진짜 원하는 재미를 정면으로 맞혀 성공한다.
- 하차 위험:
  · 업계 설명 과다 — 플랫폼 알고리즘과 작가론 설명이 길어지면 웹소설 강의처럼 보일 수 있다.
  · 숫자 반복 피로 — 조회수·선작·댓글 중계만 반복하면 장면 목표와 인물 감정이 멈출 수 있다.
  · 회귀 치트화 — 전생 지식으로 모든 선택이 쉽게 맞아떨어지면 선택과 대가가 사라진다.
  · 작중작 과다 노출 — 전생 실패작의 줄거리 설명이 길어지면 본편의 생활비·마감·경쟁 압박이 약해진다.
- 결제 압박 포인트:
  · 다음 지표 확인 — 회차 끝마다 추천, 선작, 댓글, 랭킹, 편집자 연락 중 하나가 열리기 직전에서 끊는다.
  · 경쟁 선점 대응 — 경쟁 작가가 비슷한 소재를 먼저 치고 나갈 때, 강도윤이 표절 없이 어떤 구조적 반격을 할지 궁금하게 만든다.
  · 실패 기억 회수 — 전생에 망한 결정적 선택이 이번 생에서 다른 선택으로 바뀌는 순간을 다음 화 보상으로 건다.
  · 독자 반응의 후폭풍 — 첫 고정 독자/댓글러의 작은 반응이 원고 수정, 편집자 태도, 다음 연재 전략을 바꾸게 만든다.
- 피하고 싶은 것 (사용자 회피): 표절 미화, 독자 기만, 실제 작가·플랫폼 실명 저격성 전개, 회귀 지식 단독 해결, 경쟁자 비열한 짓밟기, 댓글/조회수 반복 중계, 작중작 장문 해설, 업계 강의체, 과도한 메타 밈, 감정 과잉 신파, 보고서식 문장.
- 어휘 고정 / 금지 어휘: 제목 후보 《이번 생은 독자를 믿는다》, 《망한 작가가 회귀했다》, 《편집자였던 내가 다시 쓴다》는 후보로 보존. 실제 작가·플랫폼을 특정하게 보이는 실명 저격성 표현 금지.
```

### §A-3. 7번 전체 회차 구조화 프롬프트용 카드 (→ 5·6번 거쳐 7번에서 사용)

```markdown
- 결제 압박 포인트:
  · 회차 끝마다 지표 변화, 편집자 연락, 경쟁 작가 움직임, 고정 독자 댓글 중 하나를 미해결 후크로 남긴다.
  · 전생에서 망했던 선택과 같은 갈림길을 다시 만나게 해 이번엔 무엇을 바꿀지 궁금하게 만든다.
  · 강도윤이 윤리적 선을 지키며 이기는 방법을 다음 화 보상으로 건다.
  · 작은 지표 상승이 계약·노출·랭킹·편집자 신뢰 변화로 이어지는 상승 사다리를 만든다.
- 피하고 싶은 것 (사용자 회피): 표절 미화, 독자 기만, 실제 작가·플랫폼 실명 저격성 전개, 회귀 지식 단독 해결, 경쟁자 비열한 짓밟기, 댓글/조회수 반복 중계, 작중작 장문 해설, 업계 강의체, 과도한 메타 밈, 감정 과잉 신파, 보고서식 문장.
```

## §C. 승계 안내

- §A-1 (5번용 신규) + §B-2 (5번용 승계) → 5번이 둘 다 받아 직접 사용.
- §A-2 (6번용 신규) + §B-3 (6번용 승계) → 5번이 그대로 승계 → 6번이 둘 다 사용.
- §A-3 (7번용 신규) + §B-1 (7번용 트렌드 전략 승계) + §B-4 (7번용 1번 입력 승계) → 5·6번이 그대로 승계 → 7번이 셋 다 사용.
- §B-5 (9번용) → 5~8번이 그대로 승계 → 9번이 사용.
- §B-6 signature 카드 → 5~8번이 그대로 승계 → 9·10·11번이 직접 사용.
- §B-7 anti-signature 카드 → 5~8번이 그대로 승계 → 9·10·11번이 직접 사용.
- §B-8 비교 기준 메타 카드 단독본 → 5·6·7번이 §A 카드에 통합되어 사용 + 9·10·11번 직접 사용.
- §B-9 comparison_reference_index.md 초안 → 추후 허용 시 orchestrator가 `/output/state/`에 저장.
- §B-10 signature_log.md 베이스라인 초안 → 추후 허용 시 orchestrator가 `/output/state/`에 저장.

수정: [카드명 + 잘못된 항목 + 고칠 내용]

오류가 없으면 다음 작업에서 5단계 세계관 설계로 진행한다.

## §D. 자체점검

- §B-1 입력 ↔ 콘셉트 정합성: 정상 — 독자존중형 탑티어 성공물, 성과 지표 압박, 회귀 보상감, 직업역량 증명을 모두 반영.
- §A-1 트렌드 전략 ↔ 콘셉트 반영: 정상 — 강하게 쓸 트렌드와 일부만 쓸 트렌드, 비틀 트렌드, 회피 트렌드를 분리 반영.
- 사용자 한 끗 ↔ 한 줄 판매 포인트 직접 반영: 정상 — 표절·기만 없이 독자 반응과 정면승부로 뒤집는 약속을 중심에 둠.
- 한 줄 판매 포인트 펀치 강도: 강함 — “다시 써”, “탑티어까지 뒤집는다”로 능동 동사와 단일 상승 약속 유지.
- 양산형 위험: 낮음 — 단순 회귀 성공물이 아니라 독자존중, 편집자 출신 이중 시야, 윤리적 정면승부를 차별화 축으로 고정.
- 핵심 쾌감 펀치/깊이 균형: 정상 — 지표 역전·회귀 보상·직업 역량 증명과 독자존중 감정 보상을 함께 배치.
- 사용자 회피 ↔ 콘셉트 충돌: 없음 — 표절 미화, 독자 기만, 실명 저격, 회귀 지식 단독 해결을 회피 항목으로 고정.
- 어휘 고정 / 금지 어휘 준수: 정상 — 제목 후보 보존, 실명 저격성 표현 금지.
- 차별화 개수: 정상 — §A-1 4개, §A-2 4개, §A-3 4개.
- 하차 위험 개수: 정상 — §A-1 4개, §A-3 4개.
- 결제 압박 포인트 개수: 정상 — §A-1 4개, §A-2 4개, §A-3 4개.
- 콘셉트 ↔ §A-1 (5번용) 카드 매핑: 정상.
- 콘셉트 ↔ §A-2 (6번용) 카드 매핑: 정상.
- 콘셉트 ↔ §A-3 (7번용) 카드 매핑: 정상.
- 승계 카드 보존: 정상 — 기존 카드는 수정하지 않고 원본 파일 위치를 명시해 승계.
- 상태 파일 보존: 정상 — §B-9·§B-10은 파일 저장 없이 원본 초안 승계로만 표기.
- 다음 단계 보존 메모 정리: 있음 — 5단계는 이번 작업에서 진행하지 않음.
