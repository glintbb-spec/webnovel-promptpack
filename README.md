# Webnovel Promptpack

이 저장소는 현재 **작품별 문서·초안·시놉시스·로그·검수 저장소**입니다.
자동으로 작품을 생성하거나 여러 프롬프트를 순서대로 실행하는 운영 시스템이 아니라, 작가가 프로젝트별 자료를 정리하고 회차별 정합성을 검수하기 위한 작업 공간입니다.

## 현재 목적

- 작품별 문서, 초안, 시놉시스, 카드, 검수 로그를 한 저장소에서 관리합니다.
- 회차를 작성한 뒤 연결성, 설정, 떡밥, 캐릭터 압력, 문체 일관성을 점검합니다.
- 작가가 실제 작업 중 매 회차 쉽게 갱신할 수 있도록 단순한 템플릿과 체크리스트만 보관합니다.

## 보존된 과거 시스템

`archive/v2_5_full_system/`은 과거 V2.5 전체 자동화 시스템 보존용 디렉터리입니다.
해당 디렉터리는 참고와 기록 보존을 위한 archive이며, 새 기본 운영 방식으로 사용하지 않습니다.
V2.5 전체 자동화 방식은 archive에 보존하되, 이 저장소의 현재 작업 흐름은 작품별 문서·로그·검수 중심으로 전환합니다.

## 새 작업 위치

새 작품 작업은 `projects/project_001/` 아래에서 시작합니다.

- `projects/project_001/docs/`: 작품 설정, 작업 메모, 기획 문서
- `projects/project_001/synopsis/`: 장기·구간·화별 시놉시스
- `projects/project_001/drafts/`: 회차 초안과 수정본
- `projects/project_001/logs/`: 회차별 검수 로그와 장부
- `projects/project_001/cards/`: 작품별 캐릭터 카드, 금지 카드, 시그니처 카드

빈 폴더도 Git에서 유지되도록 각 폴더에는 빈 `.gitkeep` 파일을 둡니다.

## 템플릿과 체크리스트

### `prompts/checklists/`

원고나 시놉시스를 검수할 때 쓰는 체크리스트 보관용 디렉터리입니다.
현재는 6항목 Y/N 진단 체크리스트를 제공합니다.

### `prompts/cards_template/`

반복해서 사용할 카드 템플릿 보관용 디렉터리입니다.
캐릭터 불변 원칙 카드, 금지 문체·금지 전개 카드처럼 project별로 복사해 채워 넣는 양식을 둡니다.

### `state_templates/`

project별 `logs/`에 복사해서 쓰는 빈 로그 템플릿 보관용 디렉터리입니다.
회차가 늘어날 때마다 복사본을 갱신하며 작품의 연속성과 검수 이력을 관리합니다.

## 권장 워크플로

1. Gemini/Claude로 세계관·컨셉·100화 로그라인 작성
2. Claude로 10화 단위 시놉시스 작성
3. Gemini로 화별 시놉시스 작성
4. Gemini로 초안/확장 집필
5. Codex로 회차별 연결성/설정/떡밥/캐릭터 압력 검수
6. `logs/` 파일 갱신

## 운영 원칙

- Codex는 원고 작성자가 아니라 검수자/정합성 검사자로 사용합니다.
- 작품 설정은 사용자가 정한 장르와 구상에 따라 project별로 따로 만듭니다.
- 표준 템플릿에는 특정 장르, 특정 작품 설정, 회귀/빙의/환생 같은 전제를 넣지 않습니다.
- V2.5 전체 자동화 방식은 archive에 보존하되, 새 기본 운영 방식으로 사용하지 않습니다.
