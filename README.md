# 한국 상업 웹소설 프롬프트팩 V2.5 통합본

LOCK v1.2.1 + 14파일 표준 버전. 단일 모델(Codex/Claude Code/Cursor 등) 환경에서 100화 분량 한국 상업 웹소설을 자동 제작하기 위한 프롬프트팩이다.

V2.5는 v2.0.1에서 9번 본문이 평균값으로 회귀하는 핵심 결함을 제거한 통합본이다. 1번·8번·9번·10번·11번·orchestrator를 V2.5 통합본으로 갱신했고 toc도 V2.5로 재작성. 2~7번은 v2.1 그대로.

## 무엇이 달라졌나 — V2.5 핵심 6가지

```text
1. 1번 §B/§C/§D 정량/행동 규칙 환원
   - 형용사 답변("따뜻하게", "건조하게") → 환원 강제 절차로 정량/행동 규칙 변환
   - 사용자 슬롯(8~9턴) vs AI 자동 슬롯(S4 회차길이·S8 트렌드·S9 차별화·S14 비교 메타) 분리

2. 8번 비트 시트 도입
   - §A-1 회차 카드 8슬롯 → 비트 시트 (회차당 4~5씬 × 4~5비트)
   - 1~5화 비트 시트 5개 + 초반 설계 한 번에 출력 → 사용자 1회 검토
   - 메타 용어("보상", "훅", "압박", "결제") 비트에 박지 않음

3. 9번 자기검열 루프 차단
   - 자체점검 30+ 신호등 폐기 (9번이 본문 작성 후 평균값으로 회귀하는 직접 원인)
   - 비트 시트 살 붙이기 (1비트 30~50토큰 → 본문 100~200토큰)
   - 씬 태그 <Scene N: 시그니처 재인식> 매 씬 시작 강제 (어텐션 약화 방어)
   - 본문 출력 직후 즉시 10번 호출 (게이트 폐기)

4. 10번 1화 단위 6항목 Y/N
   - 5화 누적 19항목 신호등 → 1화 단위 6항목 Y/N
   - 핵심 적색(사건 전진 실패·캐릭터 불변 위배·시그니처 위반) 1개 → 재작성
   - 비핵심 적색(안티패턴 발생·관계 압력 부재·엔딩 훅 부재) → 다음 회차 가이드
   - 위치 명시 강제 (씬 N, 비트 X.Y)
   - 비트 누락 검증 (9번 §A-1 사용한 비트 ID와 8번/11번 비트 시트 대조)

5. 11번 정적/동적 분리 + 슬라이딩 윈도우
   - 18항목 → 정적 8 + 동적 5(활성 7 캡) + 분기점 1 + 폐기 4
   - §13 분리: §13-A 캐릭터 불변(정적) / §13-B 후폭풍(슬라이딩 7)
   - §17 3줄 고정 (5화 묶음 시점에만 갱신)
   - §18 비트 시트 형식 (9번 직접 입력)
   - §8·§11·§14·§15 폐기 (signature_log.md / anti_pattern_log.md가 자동 추적)

6. 사용자 호출 정책 — 100화 약 14~15회
   - v2.0.1: 약 40~50회
   - V2.5: 기획 9~10턴 + 작성 0턴(AUTO+적색 0개) + 분기점 4턴 = 약 14~15턴
   - 적색 발생률에 따라 변동
```

## 첫 실행 가이드 (V2.5)

### 1. 깃허브 repo 준비

이 저장소는 위 폴더 구조대로 14파일을 `/prompts`에 배치한다. 각 파일명은 v2.5/v2.1 표기 없이 단순 이름으로 통일한다.

### 2. 사용자 자료 준비 (선택)

- **/source/raw_memo.md** — 작품 기본 방향·주인공·세계관·상업적 방향·문체를 빈칸 채우기식으로 적는 시작 메모
- **/reference/like/sample_like.md** — 지향 샘플 1편을 정리하는 양식. 좋아하는 문체·대사·감정선·후킹 방식을 적는다.
- **/reference/dislike/sample_dislike.md** — 회피 샘플 1편을 정리하는 양식. 싫은 문체·전개·캐릭터 처리와 금지 규칙을 적는다.

실제 운영 시 `sample_like.md`와 `sample_dislike.md`는 복사해서 작품별 파일로 늘려도 된다. 원문 자료 없으면 1번이 사용자에게 signature 메타·비교 기준 메타 8항목씩 직접 묻는다 (V2.5: 형용사 답변 시 정량/행동 규칙 환원 강제).

### 3. orchestrator 실행

Codex/Claude Code/Cursor 등 단일 모델 환경에서 `/prompts/orchestrator.md`를 컨텍스트에 부착하고 시작 신호.

orchestrator는:
- LOCK v1.2.1 + 모듈 14파일 부착 상태 검증
- v1.x·v3.0·v2.5 패치 발견 시 /archive/ 이동 안내
- 사용자에게 AUTO 모드 / STEP 모드 묻기
- 1번부터 순차 실행

### 4. AUTO / STEP 모드 (V2.5)

```text
AUTO 모드 (기본값):
- 모듈이 자체 진행, 사용자 호출은 LOCK §3-B 의무 조건만
- 100화 정상 진행 (적색 0% 가정): 약 14~15회 사용자 호출
  · 기획 9~10턴 + 작성 0턴 + 분기점 4턴
- 핵심 적색 1개 발생 = 사용자 호출 1회 추가
- 빠른 진행, 의지 박힐 자리만 묻기

STEP 모드:
- 모듈마다 모든 모드(0/A/A 마지막/C) 사용자 답변 대기
- 100화 진행: 약 100~150회 호출
- 느리지만 단계마다 검토
```

사용자가 "알아서 해" → AUTO. "단계마다 보여줘" → STEP.

### 5. 핵심 4개 게이트 (1번 단계)

LOCK §8 핵심 4개 비면 1번 모드 C 진입 금지:

1. **지향 문체** — /reference/like/ 첨부 시 1번이 추출, 부재 시 사용자가 직접 기술 (V2.5: 형용사 답변 시 환원 강제)
2. **대사/서술 비율** — 사용자 답변 필수
3. **감정 표현 수위** — 사용자 답변 필수
4. **금지 문체** — 사용자 답변 필수

## 자주 묻는 질문

### /reference/ 자료 없어도 되나?
된다. 단 9·10번 진단 정밀도가 낮아짐. 사용자가 signature 메타·비교 기준 메타를 직접 기술하면 메타 카드 단독 운용 가능.

### 100화 다 만드는데 시간은?
AUTO 모드 기준 단일 모델 환경에서 1번~8번(셋업)에 약 3~5시간. 9·10·11번 1회차 사이클에 약 30분~1시간 (V2.5 1화 단위 루프). 100화 = 100사이클이므로 작성만 약 50~100시간. STEP 모드는 2배 정도.
