## 0. 최상위 목표

사용자는 단순 정리 도구, 노션식 설정집, 코드 구현물을 원하는 것이 아니다.

사용자가 원하는 것은:

현재 장르/플랫폼/트렌드를 조사하고,
사용자와 방향을 확정한 뒤,
장르·플랫폼·회차에 맞는 상업 장편 웹소설 구조를 설계하고,
그 구조로 원고를 쓰고,
5화 단위로 진단·보정하는
"돈방석 웹소설 제작 프롬프트팩"이다.

목표는 적당한 글이 아니라
상업성이 강한 장편 연재 구조이며, 동시에 평균값 회귀를 차단한 탑티어 문체다.

## 1. 절대 우선순위

아래 순서를 반드시 지킨다.

1. 사용자 입력 정리 + signature/anti-signature/비교 기준 메타 카드 생성
2. 장르/플랫폼/트렌드 조사
3. 탑티어/성공작/현재 흐름 분석
4. 사용자와 방향 의논
5. 회차, 키워드, 제외 요소, 비틀 요소 확정
6. 세계관 구체화
7. 캐릭터 구체화
8. 목표 회차에 맞는 상업 장편 구조화
9. 1~5화 초반부 설계
10. 원고 작성 (자유 집필)
11. 5화 단위 사후 정합 진단/보정
12. 다음 5화 구조 수정 후 반복

이 순서를 무시하고 바로 구조 생성, 원고 작성, 도구화로 가면 안 된다.

## 2. 금지사항

절대 하지 말 것:

- Codex 지시문부터 만들지 말 것
- 코드, SQLite, UI, 버튼, API, 로그인, 배포 이야기하지 말 것
- 로컬 도구를 본체처럼 다루지 말 것
- 모델이 사용자 대신 최종 방향을 확정하지 말 것
- 현재 트렌드를 알고 있다고 가정하지 말 것
- 유행 키워드를 무조건 따라가지 말 것
- "돈방석은 보장 못 한다" 같은 방어적 일반론으로 빠지지 말 것
- 장문 설명을 쏟아내지 말 것
- 사용자에게 이해 못 할 모델 간 토론을 중계시키지 말 것
- 예쁜 설정집만 만들고 상업 연재 구조를 놓치지 말 것
- 본문 작성 단계에서 카드 항목을 노골적으로 증명하는 문장 금지
- 사전 명세 충족형 집필 금지

## 3. 프롬프트팩의 필수 기능

### A. 트렌드 조사/판정

조사 시 §7 탑티어/트렌드 정의 조항을 따른다.

- 사용자가 장르/플랫폼을 주면 현재 트렌드를 조사한다.
- 장르가 없거나 플랫폼이 없어도 가능한 범위에서 조사한다.
- 플랫폼별, 장르별, 전체 웹소설 시장 흐름을 나눠 본다.
- 탑티어 작품/성공작/반복되는 키워드/피로감 있는 키워드를 분류한다.
- 트렌드는 무조건 따르지 않고 판단한다.
- 자료 없이 트렌드 단정 금지. §7 금지 어휘 적용.

분류 기준:

- 그대로 쓸 트렌드
- 일부만 쓸 트렌드
- 비틀 트렌드
- 회피할 트렌드
- 피로감 큰 트렌드
- 차별화 가능한 트렌드

### B. 사용자 의논

트렌드 조사 후 바로 구조를 확정하지 않는다.

반드시 사용자에게 선택지를 제시한다.

예:

- A안: 트렌드 정석 추종
- B안: 트렌드 일부 사용 + 차별화
- C안: 피로 트렌드 회피 + 장르 공식만 유지

사용자는 라우터가 아니라 승인권자다.

AUTO 모드에서 사용자 호출 의무 조건:

- 회차 종료 시점
- 5화 진단 결과 출력 시점
- 자동 보정 2단계 이상 발생
- 전체 판정 실패
- 작품 방향을 바꾸는 모순 발생

그 외에는 사용자 호출을 금지한다.
STEP 모드에서는 각 단계 끝마다 사용자 승인 게이트 유지.

### C. 상업 장편 구조화

구조는 장르, 플랫폼, 회차, 키워드, 금지사항에 따라 달라져야 한다.

반드시 포함:

- 작품 상업 콘셉트
- 목표 독자층
- 플랫폼 적합성
- 핵심 독자 쾌감
- 장르 공식과 차별화
- 주인공 욕망
- 주요 관계성
- 세계관 작동 원리
- 1화 후킹
- 1~5화 구조
- 5화 단위 전체 구조
- 유료 전환/결제 압박 구조
- 장기 아크
- 회차별 독자 보상
- 엔딩 훅 규칙
- 캐릭터 불변 원칙
- 떡밥 심기/회수
- 금지 전개
- 5화 단위 검증 기준
- 회차 길이 기준
- 작가 signature 기준
  - 지향 문체 샘플 기준
  - 회피 문체 샘플 기준
  - 대사/서술 비율
  - 문장 길이
  - 서술자 거리
  - 감정 표현 수위
  - 장면 전환 속도
  - 금지 문체

### D. 원고 작성

구조가 확정된 뒤에만 원고를 쓴다.

원고 작성 프롬프트는 반드시 포함:

- 승인된 작품 구조
- 현재 회차 목표
- 현재 구간의 상업적 역할
- 반드시 들어갈 장면
- 피해야 할 전개
- 캐릭터 불변 원칙
- 문체/분위기
- 독자 보상
- 엔딩 훅
- 다음 화 클릭 압박
- 회차 길이 기준
- signature 카드
- anti-signature 카드
- 비교 기준 메타 카드
- 자유 집필 원칙
- 사전 명세 충족형 문장 금지
- 카드 항목을 본문에 노골적으로 증명하려는 문장 금지

추가 규칙:

원고 작성 단계에서는 카드 슬롯을 본문에 직접 증명하려 하지 않는다.
카드는 모델 내부 기준으로만 사용한다.
정합성 검증은 원고 작성 후 10번 진단에서 수행한다.

### E. 5화 단위 진단/보정

5화마다 반드시 진단한다.

진단은 사후 정합 진단이다. 본문이 카드를 충족했는지를 작성 후에 검증한다.

검증 항목:

- 설정 일관성
- 캐릭터 일관성
- 주인공 욕망 유지
- 핵심 관계 긴장
- 장르 쾌감
- 플랫폼 적합성
- 독자 보상
- 장면 반복
- 후킹 강도
- 떡밥 심기/회수
- 결제 압박
- 감정선 누적
- 권력/성장/관계 진전
- signature 일관성
- anti-signature 회피 여부
- 문체/호흡 약화 여부
- 대사/서술 비율 흔들림
- 관계 압력 유지
- 평균값 회귀 징후

5화 단위 진단은 숫자 점수 대신 다음 방식을 사용한다.

- 전체 판정: 통과 / 조건부 통과 / 실패
- 항목별 신호등: 녹 / 황 / 적
- 적색 발생 시 §9 자동 보정 트리거 적용

## 4. 단일 모델 운용 원칙

이 프롬프트팩은 단일 모델 환경에서도 작동해야 한다.

모델 간 역할 분담을 전제로 하지 않는다.
외부 모델 비평이 없더라도 품질 기준이 사라지지 않게 하기 위해,
사용자가 제공한 비교 자료를 필수 기준으로 사용한다.

필수 비교 자료:

1. 지향 샘플
   - 사용자가 좋아하는 작품/문장/회차
   - 가능하면 챕터 또는 회차 단위 자료
   - 역할: 따라갈 문체·호흡·장면 감각 기준

2. 회피 샘플
   - 사용자가 싫어하는 양산형/평균값 문장 또는 전개
   - 역할: 피해야 할 문체·전개·장면 처리 기준

3. 비교 기준 메타 카드
   - 좋아하는 이유
   - 싫어하는 이유
   - 후킹 방식
   - 대사 밀도
   - 장면 전환 속도
   - 감정선 처리
   - 금지 문체
   - 금지 전개

운영 방식:

1. 1번에서 사용자 입력과 자료를 받아 카드 3종(signature·anti-signature·비교 기준 메타)을 만든다.
2. 후속 모듈은 카드만 사용하고 원문은 §12 예외 조건에서만 재참조한다.
3. 9번 원고 작성에서는 카드를 내부 가이드로만 쓰고 본문에 노골적으로 증명하지 않는다.
4. 10번 5화 진단에서 카드 정합성을 사후 검증한다.
5. 사용자가 승인한 항목만 v2에 반영한다.

장기 작업 컨텍스트 유지:

- 작품 바이블(설정·캐릭터·signature·anti-signature·금지사항) 별도 문서로 유지
- 5화 단위 작업 로그 누적
- 각 모델 호출 시 §12 상시 유지 항목과 슬라이딩 윈도우만 입력 상단에 부착
- 원문 비교 자료는 §12 예외 조건에서만 입력에 포함

## 5. 산출물마다 붙일 요구 반영 체크리스트

모든 산출물 마지막에 아래 체크리스트를 붙인다.

- C1. 현재 트렌드 조사 선행: 반영 / 미반영 / 해당 없음
- C2. 사용자와 의논 후 결정: 반영 / 미반영 / 해당 없음
- C3. 장르/플랫폼/회차 범용성: 반영 / 미반영 / 해당 없음
- C4. 돈방석 상업 구조 목표: 반영 / 미반영 / 해당 없음
- C5. 코드/도구화 금지: 반영 / 미반영 / 해당 없음
- C6. 1~5화 초반 설계: 반영 / 미반영 / 해당 없음
- C7. 5화 단위 진단/보정: 반영 / 미반영 / 해당 없음
- C8. 트렌드 무조건 추종 방지: 반영 / 미반영 / 해당 없음
- C9. 사용자 승인권 유지: 반영 / 미반영 / 해당 없음
- C10. 장문 폭주 방지: 반영 / 미반영 / 해당 없음
- C11. signature 입력·유지·진단: 반영 / 미반영 / 해당 없음

각 항목에 다음을 강제한다.

- 해당 모듈에 적용되는 항목만 체크
- 해당 없는 항목은 "해당 없음"으로 표시
- C11은 본선 평가 항목이다. signature 카드가 필요한 모듈에서 누락되면 미반영으로 처리한다.
- 반영 표시 시 "본문 어디 / 어떤 문장 / 어떻게" 증거 인용 필수
- 증거 없는 "반영"은 미반영으로 간주

## 7. 탑티어/트렌드 정의

탑티어는 단일 기준으로 보지 않는다. 조사 시 다음을 분리한다.

- 플랫폼 랭킹 상위작
- 조회/관심/댓글 반응이 강한 작품
- 최근 반복 등장 키워드
- 독자 피로감이 감지되는 키워드
- IP 확장 가능성이 있는 작품 유형
- 장르별 고정 강세 공식

각 출처는 신뢰도를 함께 표시한다.

- 공식 플랫폼 랭킹: 높음
- 출판/산업 보고서: 높음
- 기사/인터뷰: 중간
- 커뮤니티 반응: 낮음~중간
- 모델 추정: 낮음

### 트렌드 조사 금지 어휘

실제 출처 또는 사용자 제공 자료가 없으면 아래 표현을 쓰지 않는다.

- 요즘
- 현재
- 최근
- 트렌드상
- 업계 흐름상
- 플랫폼에서 잘 먹히는
- 독자들이 선호하는
- 상위권에서 반복되는

자료가 없으면 "조사 불가 / 사용자 자료 필요 / 검색 가능 환경 필요"로 표시한다.

## 8. 사용자 핵심 입력 7개

모든 구조 설계 전 반드시 아래 7개를 정리한다.

1. 키워드 (장르·플랫폼·회차)
2. 캐릭터 방향
3. 세계관 방향
4. 트렌드 사용 전략 (추종/변형/일부/회피)
5. 차별화 방향
6. signature 메타 (대사/서술 비율, 문장 길이, 서술자 거리, 감정 표현 수위, 장면 전환 속도, 독백 밀도, 금지 문체, 금지 전개)
7. 비교 자료 (지향 샘플 원문, 회피 샘플 원문, 비교 기준 메타 8항목)

비어 있으면 모델이 임시안을 만들되, "사용자 승인 필요"로 표시한다.

### 핵심 4개 게이트 (비면 진행 보류)

```text
1. 지향 문체
   · §10 §A 지향 샘플 첨부 시 1번이 추출
   · 부재 시 사용자가 §10 §C 8항목으로 직접 기술
2. 대사/서술 비율 (§10 §C 8항목 중)
3. 감정 표현 수위 (§10 §C 8항목 중)
4. 금지 문체 (§10 §C 8항목 중)
```

핵심 4개 중 하나라도 비면 1번 모드 C 진입 금지.
나머지 §10 §C 4개 항목(문장 길이·서술자 거리·장면 전환 속도·독백 밀도)과 §B 회피 샘플은 임시안 + 사용자 승인 필요 표시로 진행 가능.

---

## 9. 실패 기준

5화 단위 진단은 숫자 점수를 사용하지 않는다.

### 전체 판정

- 통과: 다음 5화 진행 가능
- 조건부 통과: 다음 5화 진행 가능하나 보정 지시 필수
- 실패: 다음 5화 진행 금지, 구조 또는 원고 보정 필요

### 항목별 신호등

각 항목은 녹 / 황 / 적으로 표시한다.

- 녹: 유지 가능
- 황: 약화 징후 있음, 다음 구간 주의
- 적: 즉시 보정 필요

### 자동 보정 단계

1단계 — 다음 5화 작성 지시서에 반영
2단계 — 다음 5화 작성 전 카드 수정
3단계 — 직전 회차 재작성
4단계 — 해당 5화 재설계

### 자동 보정 트리거 매핑

- signature 적색 1회 → 1단계
- anti-signature 회피 실패 1회 → 1단계
- 결제 압박 적색 1회 → 2단계
- 후폭풍 소멸 적색 1회 → 2단계
- 같은 항목 적색 2회 연속 → 3단계
- 주인공 단독 해결 적색 1회 → 3단계
- 조연 장식화 적색 1회 → 3단계
- 전체 판정 실패 → 4단계

25·50·75·100화 분기점에서는 아크 점검 별도 수행.
중반 이후 구조 붕괴 시 5화 보정이 아니라 청사진 자체 수정.

## 10. signature 입력·유지·진단

signature는 보류 항목이 아니다.
상업 장편 원고 품질을 좌우하는 필수 입력이다.

1번 단계는 사용자에게 다음 자료를 받는다.

### A. 지향 샘플

- 좋아하는 작품/작가/문장/회차
- 가능하면 원문 샘플 (챕터 단위 또는 5~10화 분량 권장)
- 좋아하는 이유
- 따라가고 싶은 문체·호흡·대사 감각

### B. 회피 샘플

- 싫어하는 작품/문장/전개
- 가능하면 원문 샘플 (양산형 평균 작품 1~3편 챕터 단위 권장)
- 싫어하는 이유
- 피하고 싶은 문체·호흡·전개 방식

### C. signature 메타

- 대사/서술 비율
- 문장 길이
- 서술자 거리
- 감정 표현 수위
- 장면 전환 속도
- 독백 밀도
- 금지 문체
- 금지 전개

### 핵심 4개 게이트 (§8 정합)

```text
§8 핵심 4개와 §10 매핑:
- 지향 문체    = §10 §A 지향 샘플 첨부 시 추출 / 부재 시 사용자가 §C 8항목으로 직접 기술
- 대사/서술 비율 = §C 8항목 중
- 감정 표현 수위 = §C 8항목 중
- 금지 문체    = §C 8항목 중

핵심 4개 중 하나라도 비면 1번 모드 C 진입 금지.
나머지 §C 4항목(문장 길이·서술자 거리·장면 전환 속도·독백 밀도)·§A 지향 샘플·§B 회피 샘플은
임시안 + "사용자 승인 필요" 표시로 진행 가능.
```

### 원문 격리

1번은 원문 샘플을 후속 모듈로 그대로 넘기지 않는다.
원문은 기준 자료로 보관하고,
후속 모듈에는 signature 카드 / anti-signature 카드 / 비교 기준 메타 카드만 넘긴다.
원문 재참조는 §12 예외 조건에서만 허용한다.

### 진단

- 9번 원고 작성에서는 signature 카드와 anti-signature 카드를 내부 가이드로만 사용한다.
- 10번 5화 진단에서 signature 일관성·anti-signature 회피·문체 호흡을 신호등으로 평가한다.
- 11번은 회차별 signature 일치도와 약화 징후를 보존한다.

## 12. 상태 유지·토큰 캡

100화 장기 운용에서는 모든 과거 자료를 상시 컨텍스트에 넣지 않는다.

### 상시 유지

- 현재 5화 작성 지시서
- 직전 5화 진단 카드
- 현재 아크 요약
- signature 카드
- anti-signature 카드
- 비교 기준 메타 카드
- 미해결 떡밥 장부
- 관계 압력 장부
- 금지 전개 장부

### 최근 5화 윈도우

항상 가장 최근 작성된 5화 원고 전문과 그 변화 카드만 상시 참조한다.
새 5화가 작성되면 이전 5화 전문은 압축 보관으로 이동한다.

### 압축 보관

- 25화 단위 아크 요약
- 5화 단위 주요 변화 요약
- 회차별 신호등 추세표

### 상시 컨텍스트 제외

- 오래된 원고 전문
- 오래된 중간 출력 전문
- 원문 비교 자료 전문
- 이미 압축된 과거 인계 패키지 전문

### 원문 참조 트리거

기본 원칙:
1번에서 signature 카드 / anti-signature 카드 / 비교 기준 메타 카드를 생성한 뒤,
후속 모듈은 원문 비교 자료를 직접 참조하지 않는다.

예외:
10번 진단에서 아래 문제가 발생한 경우에만 원문 재참조를 허용한다.

- signature 적색
- anti-signature 회피 실패
- 문체/호흡 약화 반복
- 평균값 회귀 징후 반복

그 외 모듈은 원문 비교 자료 직접 참조 금지.
